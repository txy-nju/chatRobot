# 个人助理模式 — 从零配置指南

目标：机器人代理你的飞书个人账号，自动回复他人发给你的私信。

---

## 前置条件

- 已有一个飞书账号（个人版或企业版均可）
- ChatRobot 项目已克隆，能 `docker compose up -d` 跑起来
- 浏览器能访问 `http://localhost:8000`（或你的服务器地址）

---

## 第一步：获取飞书应用凭证

如果你之前已经创建过飞书应用（用于 Bot 模式），可以复用同一个应用。如果没有，按以下步骤创建：

### 1.1 创建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/)
2. 右上角点击「开发者后台」
3. 点击「创建企业自建应用」
4. 应用名称填 `ChatRobot`（或任意名称）
5. 创建完成

### 1.2 获取凭证

在应用页面左侧「凭证与基础信息」中，记录以下两个值：

```
App ID:     cli_a43e5c6ea0fad00b  ← 你的真实值
App Secret: xxxxxxxxxxxxxxxxxxxx  ← 你的真实值
```

---

## 第二步：配置应用权限

在应用页面左侧「权限管理」中，用**中文名称**搜索（不要搜 API 标识），开启以下权限：

| 搜索中文名 | 权限标识 | 用途 |
|-----------|---------|------|
| 获取与发送单聊、群组消息 | `im:message` | 读取消息 + 发送回复 |
| **以用户身份发送消息** | **`im:message.send_as_user`** | **以用户身份发送回复（必加！）** |
| 以用户身份获取单聊消息 | `im:message.p2p_msg:get_as_user` | 读取发给你的私聊消息 |
| 以用户身份获取群聊消息 | `im:message.group_msg:get_as_user` | 读取群聊中的消息 |
| 获取会话列表 | `im:chat:readonly` | 获取你的会话列表 |

> **搜索技巧**：飞书后台不支持搜 `im:message` 这样的英文标识，必须用中文。如果搜不到，先检查左侧「应用功能」→「机器人」能力是否已开启——部分权限依赖机器人能力。
>
> **批量开通**：开启权限后，如果有「批量开通」按钮，点击一下。如果提示需要管理员审核，先跳过。

---

## 第三步：发布应用（重要！）

**即使只是自己测试，也必须发布一次。** 未发布的应用调用用户级 API 会报 `The app is unavailable or inactivate in the tenant`。

1. 应用左侧 → **版本管理与发布**
2. 点击「创建版本」，填写版本号（如 `1.0.0`）
3. **可用范围**选择「**仅应用创建者可用**」
4. 点击「发布」

> 不需要全企业可见，"仅应用创建者可用"就能跑通。

---

## 第四步：配置回调地址

### 4.1 本地测试

在应用页面左侧「安全设置」→「重定向 URL」中，添加：

```
http://localhost:8000/api/oauth/callback
```

### 4.2 服务器部署

如果你的 ChatRobot 部署在服务器上（有公网域名），添加：

```
http://你的域名:8000/api/oauth/callback
```

可以添加多个回调地址，本地和服务器各一个。

---

## 第五步：修改 .env 文件

在项目根目录编辑 `.env` 文件（从 `.env.example` 复制而来）：

```bash
# ── 下面两项填入第一步获取的凭证 ──
FEISHU_APP_ID=cli_a43e5c6ea0fad00b       # 改成你的真实 App ID
FEISHU_APP_SECRET=你的AppSecret           # 改成你的真实 App Secret

# ── LLM 配置（必填）──
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o
OPENAI_API_KEY=sk-your-api-key            # 改成你的真实 API Key

# ── 以下可选 ──
# 本地测试不需要填，会自动用 localhost
# 服务器部署时取消注释并改成真实地址：
# FEISHU_REDIRECT_URI=http://你的域名:8000/api/oauth/callback

# Bot 模式配置（群聊自动回复，和飞书事件订阅一起用）
FEISHU_VERIFICATION_TOKEN=               # Bot 模式不用可以不填
```

---

## 第六步：获取 LLM API Key

这一步和 Bot 模式一样。任选一个 LLM 供应商获取 API Key：

**OpenAI:**
1. 打开 [platform.openai.com/api-keys](https://platform.openai.com/api-keys)
2. 创建新 Key → 复制 → 填入 `.env` 的 `OPENAI_API_KEY`

**DeepSeek（更便宜）:**
1. 打开 [platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys)
2. 创建新 Key → 复制
3. `.env` 中设置：
```bash
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-chat
DEEPSEEK_API_KEY=sk-your-key
```

## 第七步：启动服务

```bash
# 拉取最新代码
git pull

# 重新构建并启动
docker compose up -d --build

# 确认运行正常
curl http://localhost:8000/api/health
# 返回 {"status":"ok"} 即正常
```

## 第九步：测试

### 9.1 找个人给你发私信

在飞书中，让一位同事或朋友直接给你发一条私聊消息（不是发到群里，不是发给机器人）：

```
同事 → 你（私聊）: "张三，明天几点开会？"
```

### 9.2 观察自动回复

10 秒内，同事会收到一条以**你的身份**发送的回复：

```
你 → 同事（私聊）: "明天上午10点，3楼会议室。"
```

### 9.3 在管理后台确认

刷新个人助理页面，你会看到「今日回复」数字变成了 `1`。

## 第十步：自定义回复风格

机器人的回复内容由 Skill 控制。在管理后台「📋 Skill 管理」中编辑活跃的 Skill：

```markdown
# Skill: 张三的工作分身

## 角色
你是张三，某科技公司的后端工程师。你收到同事发来的私信，
需要以张三的身份简洁回复。

## 口吻
直接、简洁，偶尔带点幽默。像同事之间的正常聊天。

## 回复规则
1. 如果不知道答案，回复"我查一下，稍等"
2. 回复控制在 3 句话以内
3. 如果对方问时间/地点，确认后再回复

## 知识库
- Q: 明天几点开会
- A: 明天上午10点，3楼会议室
- Q: 你的工位在哪
- A: 5楼西侧，靠窗第二个
```

保存后，机器人立刻使用新的 Skill 回复。

## 常见问题

### Q: Token 过期了怎么办？

2 小时内会自动刷新。如果 refresh_token 也过期了（约30天后），打开管理后台 → 个人助理 → 点击「🔄 重新授权」，30 秒搞定。

### Q: 怎么关掉自动回复？

在管理后台的个人助理页面，目前没有暂停按钮。**临时关闭方法**：重启容器时不启动轮询（下个版本会增加 UI 开关）。或者直接把 `FEISHU_APP_SECRET` 改错让授权失效。

### Q: 我不想回复某个人怎么办？

修改活跃 Skill 的回复规则，添加：
```
5. 如果消息发送者署名是"王五"，回复"我不在，稍后回复"
```
或者把对方关键词加到行为配置的黑名单里。

### Q: 怎么区分 Bot 模式和个人助理模式？

| | Bot 模式 | 个人助理模式 |
|------|----------|-------------|
| 触发 | 发给群里的机器人 | 发给你个人的私信 |
| 发送者显示 | 机器人名称 | 你的真实姓名 |
| 需要配置 | 飞书事件订阅 + 公网URL | OAuth 授权（本文档） |
| 可以同时用吗 | ✅ 可以 | ✅ 可以 |

两种模式共享同一个 Skill 和 LLM 配置。

### Q: 本地测试没有公网 URL 可以吗？

可以。OAuth 授权用的是浏览器跳转，`localhost` 完全没问题。授权完成后飞书会把 code 回调到 `localhost:8000`。只有 Bot 模式（事件订阅）才需要公网 URL。
